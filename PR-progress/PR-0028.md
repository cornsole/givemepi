PR-0027 안정화 결과:

- production StorageMergeCoordinator와 bounded async writer/reader 연결 완료
- NodeLifecycle durable publication, 실패·복구, 종료 계약 검증 완료
- ProgressTracker에 async I/O queue/active/completed/failed telemetry 연결
- sync/async forced out-of-core benchmark baseline 기록
- 전체 CTest 64개 통과

PR-0028 범위:

1. Concurrent I/O execution
   - worker 수 1/2/4/8 비교
   - queue capacity와 backpressure 대기 시간 측정
   - multi-chunk read/write throughput 검증

2. Compression backend optimization
   - none/LZ4 크기·처리량 비교
   - codec CPU 비용과 reload 비용 분리 측정

3. NUMA/Huge Pages optimization
   - target platform 확인 후 선택 적용
   - peak RSS와 end-to-end 결과 회귀 검증

PR-0028은 PR-0027의 chunk identity, codec bytes, lifecycle, progress,
synchronous fallback 계약을 변경하지 않는 조건으로 진행한다.

초고성능화의 단계별 측정 기준과 acceptance gate는
`docs/PR-0028-HIGH-PERFORMANCE-ROADMAP.md`에 정리한다. 먼저 병목을
분해 측정하고, 그 다음 concurrent I/O, codec/data movement, NUMA/Huge
Pages 순서로 최적화한다.
