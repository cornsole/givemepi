PR-0025 남은 작업 구현 순서는 다음이 적절합니다.

  1. Chunk Index 계약 확정
      - VersionedChunkIndex와 실제 chunk 파일 연결
      - duplicate identity 거부
      - range overlap 검증
      - stored size/checksum 집계
      - index atomic 저장 및 재시작 reload

  2. ChunkStore와 ChunkCodec 통합
      - 현재 임시 metadata sidecar 구조를 canonical ChunkCodec 기반으로 통일
      - computation identity, range, tree level을 reload 시 검증
      - checksum·payload·format version 검증 일원화

  3. StorageManager 구현
      - store
      - load
      - contains
      - remove
      - snapshot
      - planEvictions

     Binary Splitting은 파일 경로와 codec 세부사항을 직접 알지 않도록 유지합니다.

  4. Memory budget 및 eviction planner 고도화
      - resident/stored byte 정확한 accounting
      - merge에 필요한 node eviction 금지
      - 동일 입력에 대한 deterministic ordering
      - store 실패 시 resident 값 보존
      - planner 단위 테스트 보강

  5. Progress/config 연동
      - storage directory
      - memory budget
      - target chunk size
      - compression
      - resident bytes
      - stored bytes
      - chunk count

  6. 재시작·무결성 통합 테스트
      - store → 프로세스 종료 → index reload → load
      - checksum 손상
      - truncated chunk
      - 잘못된 identity/range
      - atomic write 실패
      - checkpoint codec 값과 P/Q/T equality

  7. 대형 데이터 및 성능 검증
      - 대형 GMP P/Q/T round-trip
      - 100MB 이상 synthetic chunk
      - in-memory 대비 store/reload throughput
      - GCC/Clang
      - ASan/UBSan
      - 정적 분석

  현재 상태는 “기초 storage 계층과 오류 수정 완료”이며, 다음 구현 우선순위는 Chunk Index 통합 → ChunkStore/ChunkCodec 통합 → StorageManager입니
  다. PR-0025 문서의 실제 Binary Splitting eviction 연결은 기존 계획대로 PR-0026 범위로 유지하는 것이 맞습니다.
