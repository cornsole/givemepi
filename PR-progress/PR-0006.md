## 제안

**PR-0006에서는 `Stress Test`까지만 구현하고, `Benchmark`는 다음 PR로 넘기는 것을 추천합니다.**

이유는 현재 프로젝트 규칙이

* 작은 PR
* 항상 빌드 가능
* 항상 테스트 통과

이기 때문입니다.

그러면 PR-0006의 범위가

> **"Reference Queue를 Vyukov MPMC Queue로 교체하고, 기존 SchedulerTest + Stress Test를 모두 통과시킨다."**

로 명확해집니다.

**가장 먼저 할 일은 구현이 아니라 `DECISIONS.md`에 "Vyukov Bounded MPMC Queue를 채택한다"는 ADR를 추가하는 것입니다.** 그 결정이 있어야 이후 구현이 문서와 일치하게 진행됩니다.

1. DECISIONS.md
    - Lock-Free Queue로 Vyukov Bounded MPMC Queue 채택
    - Reference Queue는 개발 전용으로 병행 유지

2. CHECKLIST 확인
    - 이번 목표는 Lock-Free Queue 하나임을 확인

3. 구조 설계
    - Queue 인터페이스(IQueue 등)를 둘지
    - ReferenceQueue / LockFreeQueue 파일 구조 결정

4. create-scheduler-structure.sh 수정(필요 시)

5. 코드 구현 시작
