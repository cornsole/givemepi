  1. 정밀도 정책

  - requested digits는 0보다 커야 함
  - guard digits 최소값은 16, 기본값은 32
  - working digits는 requested + guard로 계산
  - 모든 파생 크기는 overflow를 검사
  - host floating-point를 정책 계산에 사용하지 않음
  - Chudnovsky digits-per-term 하한 14.181647 사용
  - ceiling term count에 safety term 1개 추가
  - 예상 binary precision은 log2(10) 상한 3.322와 64-bit headroom 사용
  - 1자리·기본값·단조성·10억 자리·overflow 경계 테스트

  2. GMP 정수 최종화 연산 보강

  - power-of-ten 연산과 exponent 범위 검증 추가
  - non-negative floor integer square root와 음수 거부 추가
  - signed floor division과 zero divisor 거부 추가
  - 오류 발생 시 기존 값 보존 검증
  - guard scale에서 fixed-point sqrt와 division 수행

  3. End-to-end 계산 API

  - digits에서 immutable precision plan 생성
  - sequential 또는 staged-parallel P/Q/T 실행
  - guarded integer fixed-point sqrt와 division 적용
  - guard 제거 전 half-up rounding 적용
  - normalized 3.<requested digits> 문자열 생성
  - precision identity와 decimal을 result로 반환
  - 1·2·10·20·50·100자리 rounded known digits 검증

  4. 정확성 및 성능 검증

  - 여러 precision 경계의 known digits 검증
  - sequential/parallel output 완전 일치
  - split·finalize·format 단계별 benchmark
