1. Scheduler 수명주기 계약 확정

  - Stopped / Running / Stopping 상태 구분
  - start()와 stop() 동시 접근 보호
  - stop 시작 후 신규 제출 거부
  - 여러 번 호출해도 안전한 start/stop
  - 재시작 지원 여부 명시 및 테스트

  2. Drain shutdown 구현

  - stop 전에 승인된 작업은 모두 완료
  - 모든 worker에 먼저 종료 요청한 뒤 join
  - global/local queue가 빌 때까지 worker 실행
  - stop 반환 후 accepted handle은 반드시 terminal 상태
  - 중단된 채 Pending으로 남는 handle 방지

  3. Global/local queue 역할 정립

  - 외부 스레드 제출 → bounded global queue
  - worker 내부에서 생성한 작업 → 해당 worker local queue
  - 공개 API는 기존 submit(Task) → TaskHandle 유지
  - worker context 식별 방식 추가
  - global queue full이면 invalid handle 반환

  4. LockFreeQueue 정확성 보강

  - zero capacity 처리
  - full과 contention 상태 구분
  - producer/consumer CAS 재시도 로직 보강
  - 순환 및 sequence wrap 동작 검증
  - 불필요한 소스 문구 제거

  5. 직접 동시성 테스트 추가

  - 다중 producer / consumer
  - 작업 유실·중복 실행 검사
  - bounded capacity 및 full rejection
  - queue 재사용과 여러 번의 wrap
  - 즉시 stop해도 accepted task 전체 완료
  - stop 중 제출 거부
  - start/stop 반복 동작

  6. 실제 work-stealing 테스트

  - 한 worker의 local queue에 child task 집중
  - 다른 worker가 task를 훔쳐 실행하도록 구성
  - 둘 이상의 worker/thread에서 실행됐는지 확인
  - 단순 완료 테스트와 steal 동작 테스트 분리
