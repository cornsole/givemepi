  1. Checkpoint 계산 identity와 경계 확정

  - algorithm id·arithmetic format version 포함
  - requested·guard·working digits와 term count 포함
  - block의 [start, end)와 tree level 포함
  - cutoff·worker count·tasks-per-worker는 compatibility identity에서 제외
  - timing과 queue capacity는 optional provenance로만 취급
  - resume·trust·quarantine은 PR-0022로 유지

  2. Canonical GMP serialization

  - signed magnitude encoding 사용
  - zero는 zero sign과 empty magnitude로 고정
  - positive·negative magnitude는 minimal big-endian bytes
  - leading zero와 invalid sign 조합 거부
  - zero·양수·음수·대형 정수 round-trip 테스트

  3. Versioned P/Q/T block codec

  - magic·version·header size·flags 정의
  - 모든 fixed-width integer는 explicit big-endian
  - native C++ struct dump 금지
  - computation identity와 range·level metadata encoding
  - P/Q/T sign·length·payload encoding
  - checked size arithmetic와 deterministic bytes

  4. CRC32C checksum metadata

  - checksum algorithm id와 length를 version 1 header에 포함
  - checksum field를 zero 처리한 canonical header와 payload 전체를 보호
  - fixed fixture와 single-bit mutation 테스트
  - checksum trust validation은 PR-0022에서 수행

  5. Atomic file commit

  - destination과 같은 directory에 unique temp 생성
  - short write 처리 후 flush·file sync·close
  - atomic rename 후 parent directory sync
  - 기존 final file은 commit 성공 전까지 보존
  - fault point별 interrupted-write 테스트

  6. CheckpointStore와 deterministic filename

  - codec과 filesystem storage 책임 분리
  - stage coordinator만 synchronous store 호출
  - scheduler worker 내부 disk I/O 금지
  - 동일 identity·range·level은 동일 final filename 사용

  7. Versioned manifest

  - computation identity와 block entry 목록 저장
  - range·level·filename·payload size·checksum·completion state 포함
  - range와 level 기준 canonical sorting
  - manifest도 동일 atomic commit protocol 사용
  - insertion order와 무관한 byte equality 테스트

  8. 회귀 및 마무리

  - in-memory block byte-stable round-trip
  - filesystem block·manifest round-trip
  - 기존 final 보존과 temp file isolation 검증
